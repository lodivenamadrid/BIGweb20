HTML Forms

When we look at a website that requires a user’s input such as filling out one’s name etc. We as user’s don’t necessarily think that a website uses this input in an HTML form that involves coding to relay or gain profit or information. According to Internetingishard.com, “forms are the “money pages.” They’re how e-commerce sites sell their products, how SaaS companies collect payment for their service, and how non-profit groups raise money online (James).”

Using an HTML form can help with feedback to a developer’s website or to provide some information about yourself to the web server. According to GeeksforGeeks.com, how a form is presented through HTML on a website is by “a document which stores information of a user on a web server using interactive controls. The elements used in an HTML form are check box, input box, radio buttons, submit buttons etc. (Geeks For Geeks)”

One thing I found interesting and also useful with HTML forms is that “many companies measure the success of their website by the effectiveness of its forms because they answer questions like “how many leads did our website send to our sales team?” and “how many people signed up for our product last week? (James)” I found this interesting and mainly useful to use when considering how to develop your webpage because it will give you criticism on how well your website is really doing and what you could do to revise it if necessary.

HTML forms should be an important consideration when developing and creating your webpage. The necessity and importance of providing a form to users, can make your site user-friendly or in return receive great feedback from users.
